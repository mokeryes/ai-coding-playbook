# SPEC-Driven Agentic Coding 工作流

> 阶段 1 占位文件。**阶段 2 会从 wework-opus `CLAUDE.md` § 11 + § 14 抽取并扩充**。

## 核心思想

```
先定规格,再写代码;
边写边验,提交留痕;
AI 执行,人来把关。
```

## 4 阶段流程(摘要)

1. **方案确认** — AI 写 `app/modules/<X>/SPEC.md` → 用户审核 → 明确"开始写代码"
2. **代码生成** — 严格按 SPEC 实现,不擅自越界
3. **AI 自验** — 语法 / import / schema 自跑(命令视语言)
4. **联调验证** — 按 SPEC 的"验收标准"逐项核对

## 紧急沟通规则(摘要)

5 类必须停下询问:
1. 业务逻辑歧义
2. 跨模块边界争议
3. 性能/成本权衡
4. 安全敏感操作
5. 与本文档冲突的需求
