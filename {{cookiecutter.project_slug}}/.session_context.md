## 当前阶段(SPEC-driven)

> 由 AI 在阶段过渡时维护;用户可以手改作为修正。
> 仅当 `active_spec` 非 null 时,AI 每次回答末尾追加进度块。
> 详细规范见 `docs/workflows/spec-driven.md` §"进度可视化"。

- **active_spec**: `null`
- **major**: `null`
- **sub**: `null`
- **flag**: `normal`
- **retry_count**: `0`

> 字段含义:
> - `active_spec`:当前 SPEC 文件路径(如 `app/modules/<X>/SPEC.md`),不在 SPEC 流程则 null
> - `major`:1-6(方案 / 代码 / 自验 / 联调 / 收尾 / 提交)
> - `sub`:子步骤编号(如 `1.3`、`2.1`)
> - `flag`:`normal` / `interrupt` / `rollback` / `retry`
> - `retry_count`:自验失败重试计数(≥ 3 强制升级 interrupt)
